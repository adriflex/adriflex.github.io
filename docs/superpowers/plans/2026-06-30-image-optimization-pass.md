# Image Optimization Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce portfolio/blog image weight while preserving useful lightbox zoom quality.

**Architecture:** Keep original source images outside the published `public/images` folder and publish web-ready derivatives from the same filenames updated to `.webp`. Markdown and Astro references will point at optimized files, so GitHub Pages serves smaller assets without runtime image processing.

**Tech Stack:** Astro 5, static GitHub Pages build, Node.js, Sharp.

## Global Constraints

- Preserve zoom quality: cap standard article images at 2400 px wide, with a 3200 px cap for panorama/hero images.
- Preserve originals before replacing public assets.
- Use WebP quality 86 for still images.
- Do not optimize GIF/WebM in this pass.
- Verify with an image weight audit and `npm run build`.

---

### Task 1: Backup and Generate Optimized Images

**Files:**
- Create: `tools/optimize-images.mjs`
- Create: `.image-originals/`
- Modify: `public/images/`

**Interfaces:**
- Consumes: images in `public/images/`
- Produces: optimized `.webp` still images and a JSON mapping from old image paths to new image paths

- [x] **Step 1: Create a Sharp-based script**

Create `tools/optimize-images.mjs` that copies original still images to `.image-originals/`, converts `.png`, `.jpg`, and `.jpeg` files to `.webp`, caps width at 2400 px by default and 3200 px for wide hero/panorama images, and leaves existing `.webp`, `.gif`, and `.webm` files untouched.

- [x] **Step 2: Run the script**

Run: `node tools/optimize-images.mjs`
Expected: optimized `.webp` files in `public/images/`, originals backed up in `.image-originals/`, and a mapping file for reference updates.
